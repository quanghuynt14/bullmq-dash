# Code of Conduct

## Our Standard

This project is for people building and operating BullMQ systems. Keep
discussion respectful, specific, and focused on improving the software.

Examples of expected behavior:

- Use welcoming and inclusive language.
- Respect different experience levels and operating contexts.
- Give constructive feedback tied to code, behavior, or documented goals.
- Accept responsibility for mistakes and correct them quickly.

Examples of unacceptable behavior:

- Harassment, insults, or personal attacks.
- Public or private abuse, including unwanted attention.
- Publishing someone else's private information.
- Repeated disruption after maintainers ask for a change in behavior.

## Enforcement

Maintainers may remove comments, close issues, block accounts, or take other
reasonable action when behavior harms the project.

Report conduct concerns through GitHub Issues if public disclosure is safe, or
contact the repository owner privately through GitHub when it is not.
